Responsive Webpages

Responsive Webpages are one of the most influencial and one of the easiest items a web developer can control. The majority of web users use mobile to access the web, therefore, a website NEEDS to be responsive.

How did they do?

The image above is from Potter's Bakery. Their website on a desktop screen looks fairly good, however, when you switch to a mobile perspective, it becomes unuasble. If I am a mobile user and see this page, why go through the trouble of trying to find my way through this page when I can find another bakery somewhere else with a better viewing experience? Beyond that, Google is going to index rank this website worse than someone who DOES focus on responsive webpages.

404 Error Pages

404 Error Pages can be overlooked pretty easily because as long as the user follows your site as you intend, nothing bad should happen, right? There is always ways a user will use the website other than intended, or there could even be a issue that wasn't identified. Nonethless, having a custom 404 error page will let the user know something bad happen, and allows you to guide their next step.

How did they do?

The photo above is from Endorphin Fitness. They do a awesome job with their custom 404 error page. They let the user know something happened, however, they have a search bar and the nav bar present to allow the user to get back on track. This allows for the opportunity for the user to stay on the site for a longer period of time.

Robots.txt

A robots.txt file is a file that only web crawlers will view. A normal user will likely never encounter this file. This file tells the crawlers which parts of the site NOT to index. This is useful for not allowing an admin login or a cart link from being a option (index) to choose from when using a search engine.

How did they do?

The photo above is from SportClips. SportClips does a good job telling ALL search engine crawlers not to index certain items, such as their checkout or admin login pages. Furthermore, they also link their sitemap.

Sitemap for Crawlers (XML)

A Sitemap for Crawlers can be extremely useful when a website is large and complex in size. For example, a hobbyist's website may not want to look in sitemaps, however, companies like Planet Fitness would want to look into creating one. This sitemap allows crawlers to better understand your site, such as the relationship between pages, and where certain information is. Alongside that, crawlers work more efficiently when there is a sitemap.

How did they do?

The photo above is from Mill Inn. While the Mill Inn is not a complicated website, they provided a sitemap, so, crawlers will work more efficiently. They used a site to auto generate this file, and properly lays out the information that would be needed for crawlers. Overall, they seem to created a valid sitemap for crawlers

Page Load Speed

If a page loads fast, a user will see it faster and be less inclined to go somewhere else. If a user has to sit on a loading page screen for even a 5 seconds or more, there is a high probability they will leave and look somewhere else. A page that loads fast will not only retain these users, but rank better in search engine ranks.

How did they do?

The photo above is from Grace Ella Photograpghy. These page is a work in-progress, however, the images and page load within a 1-2 seconds after going to their site. This means the page loads relatively fast. There could be some work done for some photos as they loaded slower then others, however, its good overall.

Page Titles

Page titles are simple, but yet the most important thing when a visitor uses a search engine. Not only will it be the first impression on the user, it also needs to cater towards the users needs.
